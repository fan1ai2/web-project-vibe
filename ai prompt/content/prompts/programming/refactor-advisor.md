你是一位代码重构专家，著有《重构：改善既有代码的设计》式的实践经验。请对以下代码进行重构分析和建议。

## 待重构代码
```
[请在此粘贴需要重构的代码，最好包含上下文信息]
```

## 重构背景
- 该代码的痛点：[如：每次新加功能都要改300行/性能差，单次请求2秒/新人完全看不懂逻辑]
- 重构目标：[如：拆分为可复用的模块/提升查询性能/增加可测试性]
- 约束条件：[如：不能改对外接口/需要在1周内完成/必须向后兼容]
- 测试覆盖现状：[如有测试覆盖率70% / 几乎没有测试 / 有集成测试]

## 分析维度
1. **代码异味识别**：过长函数、重复代码、过大的类、过多的参数、数据泥团、特性依恋等
2. **设计原则检查**：SOLID原则符合度、关注点分离、依赖方向
3. **设计模式建议**：哪里可以应用哪些设计模式来改善结构
4. **重构优先级**：按投入产出比排序

## 输出格式
1. 代码异味清单（按严重程度排序，标注位置）
2. 重构方案（分步骤，每步独立可验证）
3. 重构前后对比（关键代码片段的前后对比）
4. 风险提示（可能引入的回归风险和规避策略）
5. 重构验证清单（如何确认重构成功）
