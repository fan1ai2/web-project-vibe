你是一位前端架构师兼UI设计顾问，精通响应式设计和移动端适配。请为以下项目提供移动端适配方案。

## 项目信息
- 项目类型：[如：电商H5活动页 / 企业官网 / Web应用管理后台 / 内容型博客]
- 当前设计稿基准：[如：375px（iPhone SE）/ 390px（iPhone 14）/ 1440px（桌面端）]
- 需要支持的设备范围：[手机/平板/桌面，指定最小支持的屏幕宽度，如320px]
- 关键页面类型：[首页/列表页/详情页/表单页/数据图表页]

## 适配方案要求
1. **断点策略**
   - 推荐断点系统（至少4个断点：mobile small / mobile large / tablet / desktop）
   - 每个断点的布局策略（单列/双列/多列/侧边栏显隐）
   - 容器宽度处理：fixed-width vs fluid

2. **交互适配**
   - 触摸目标最小尺寸（44×44px WCAG标准）
   - hover状态的移动端降级方案
   - 手势操作规范（滑动/长按/双指缩放）
   - 虚拟键盘弹出时的布局处理

3. **性能优化**
   - 图片适配方案（srcset/sizes/picture元素）
   - 移动端字体大小基准
   - 按需加载组件/内容策略

4. **导航模式**
   - 移动端导航方案选择：汉堡菜单/底部Tab栏/浮动按钮
   - 面包屑在移动端的展示方式

## 输出
- 断点定义表（含示例CSS media query）
- 每个关键页面的多端布局示意描述
- 移动端适配检查清单（测试时需要验证的10个要点）
