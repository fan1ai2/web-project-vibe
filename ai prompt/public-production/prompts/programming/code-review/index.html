代码审查提示词

编程开发 高级
提示词内容 
你是一位资深技术负责人，以严谨、建设性的态度进行代码审查。请对以下代码进行全面的Code Review。

## 审查的代码
```
[请在此粘贴需要审查的代码，建议控制在200行以内]
```

## 审查背景
- 项目类型：[如：电商后端API / 移动端SDK / 数据管道]
- 该代码的职责：[如：订单支付回调处理 / 用户认证中间件 / 数据导出模块]
- 性能要求：[如：高并发1000 QPS / 日常批处理 / 毫秒级响应]
- 团队技术栈：[如：团队主要用Python，正在向Go迁移]

## 审查维度（请逐一覆盖）
1. **正确性**：逻辑是否有误，边界条件是否处理，是否有潜在的空指针/未定义行为
2. **安全性**：是否存在注入风险、敏感信息泄露、权限绕过、不安全的依赖
3. **性能**：是否存在N+1查询、不必要的循环、内存泄漏、大对象拷贝
4. **可维护性**：命名是否清晰、函数是否过长、耦合度是否过高、测试覆盖是否充分
5. **最佳实践**：是否遵循语言/框架惯例，错误处理是否得当，日志是否合理

## 输出格式
- 总体评价（1-5星，一句话总结）
- 严重问题（Must Fix）：列出每个问题的位置、原因、修复建议
- 改进建议（Nice to Have）：按优先级排列
- 代码亮点（做对了什么，值得保留和推广）
