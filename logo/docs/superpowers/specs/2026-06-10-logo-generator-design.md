# Logo 自动生成器 — 产品设计规格书

## 一、产品概述

一个商标自动生成的网页工具。用户通过五维度关键词选择器描述需求，系统拼合 prompt 提交 AI 生成 logo，同时提供高质量品牌案例墙作为灵感入口。

**目标用户**：创业者、小企业主、自由设计师，需要快速获得品牌 logo 创意

**核心差异**：不是冷冰冰的 AI 工具，而是一个像设计杂志 + 独立工作室作品集的创意体验

---

## 二、页面架构 & 路由

| 路由 | 用途 | 渲染方式 |
|------|------|----------|
| `/` | 案例灵感墙 + CTA 入口 | Hugo 静态生成 |
| `/generator/` | 仪表盘：筛选面板 + 实时预览 + 生成 | Hugo 壳 + JS 交互 |
| `/cases/<slug>/` | 单案例完整参数 + prompt + 大图 | Hugo 静态生成 |
| `/keywords/<组合>/` | SEO 落地页，特定关键词组合的案例集合 | Hugo 静态生成 |

## 四、关键词筛选维度（五维）

| 维度 | 控件 | 选择 | 说明 |
|------|------|------|------|
| 行业/品类 | 图标网格 4×N | 单选 | 每项配 icon，选中高亮边框 |
| 视觉风格 | 胶囊标签流 | 单选 | 横向排列，选中填充主色 |
| 配色方案 | 色块圆点阵 | 多选(≤2) | 渐变色圆，hover 放大 |
| 字体气质 | 字体样本卡 | 单选 | 对应字体写 "Ag" |
| 构图类型 | 示意图标 | 单选 | 简笔示意 icon/文字/徽章/组合 |

附加条件：自由文本区，placeholder "比如：不要渐变、logo 里面藏一只猫..."

## 五、视觉设计方向 — 创意画廊 B

**气质**：暖调设计工作室 × 独立杂志 × 纸张肌理感

### 设计令牌

```
颜色：
  背景:      #FAF8F5  暖纸白
  卡片:      #FFFFFF
  主文字:    #1E1E1E
  次文字:    #6B6560
  强调色:    #D4532A  陶土红（按钮、选中态）
  辅助强调:  #2D5F7C  靛蓝（链接、标签）
  成功:      #3A6B5C  森林绿
  线条:      #E8E3DD
  预览画布:  #1E1E1E  深色底展示 logo

字体：
  品牌标题: Playfair Display (衬线)
  UI 正文:  Inter / IBM Plex Sans
  代码:     JetBrains Mono
  中文:     Noto Serif SC (标题) + Noto Sans SC (正文)

间距：8px 基准栅格
圆角：4px 卡片 / 8px 按钮 / 0 画布
```

### 动态渲染效果

- 关键词选中：胶囊从线条边框过渡到填充色，带 300ms ease-out
- 预览区模板切换：SVG 形状间做 crossfade 过渡（300ms），不依赖 morphing 以兼容更多浏览器
- 生成中：预览区骨架屏 + 逐帧笔触动画（模拟"AI 在画"）
- 案例墙卡片：hover 上浮 4px + 阴影加深 + 图片微缩放
- 页面滚动：案例墙卡片逐个淡入（Intersection Observer + CSS animation）
- 生成完成：logo 从模糊到清晰的 reveal 动画

## 六、案例墙 & 预览区

### 案例墙（首页）
- 顶部：品牌标题 + 描述文案 + "开始创作" CTA
- 主体：Masonry 瀑布流卡片网格
- 卡片内容：logo 图 + 风格标签 + 行业名
- hover：上浮 + 阴影 + 图片缩放
- 顶部轻量风格过滤条
- 图片 `loading="lazy"` + `srcset` avif/webp

### 实时预览区（生成器）
- 空状态：虚线方框 + 引导文案
- 选中态：即时切换示意模板
- 生成中：骨架闪烁 / 笔触动画
- 完成态：大图展示 + 下载/分享/重生成按钮

---

## 七、案例详情 & SEO 页面

### 案例详情 `/cases/<slug>/`
- 左侧大图，右侧参数面板
- 五维度标签（可点击跳转关键词专题页）
- 完整 prompt 代码块（一键复制）
- "套用参数去生成器" 按钮
- 底部相关案例推荐

### SEO 关键词着陆页 `/keywords/<组合>/`
- Hugo 从 data 文件静态生成
- 标题格式：`「关键词组合」Logo 设计灵感`
- 结构化数据：CollectionPage + ItemList schema
- Breadcrumb：首页 > 关键词专题
- 该组合下所有案例卡片网格

---

## 八、数据架构

```
data/cases/brand-logos.yaml (51 条种子数据)
  → Hugo data system
    → /              (案例墙)
    → /cases/<slug>/ (单案例详情)
    → /keywords/*    (SEO 着陆页)
    → /generator/    (JS 消费 window.__CASES__ JSON)
```

- 构建时 YAML → JSON 注入生成器页
- 案例详情和关键词页纯静态，确保 SEO
- 未来 AI 生成案例追加入 YAML，重建即静态化

---

## 九、技术栈

- **Hugo**：页面骨架、路由、SEO 元标签、结构化数据、静态生成
- **纯 CSS**：设计令牌（CSS Custom Properties）、布局（Grid/Flexbox）、动画
- **纯 JS**：仪表盘状态管理、筛选联动、预览切换、模板渲染
- 零 JS 框架依赖

## 十、响应式策略

| 断点 | 布局 |
|------|------|
| 320-767px | 单列堆叠，筛选变底部 Sheet，历史栏隐藏 |
| 768-1023px | 筛选面板折叠为 icon 栏，预览区占主体 |
| 1024-1439px | 标准三栏，280+flex+200 |
| 1440px+ | 最大宽度 1440px 居中，两侧留白 |

---

## 十一、SEO 策略

- 每个案例详情页独立 title/description/OG 标签
- 关键词组合页生成 CollectionPage + ItemList 结构化数据
- Logo 图片使用 ImageObject schema
- sitemap.xml 自动包含所有案例和关键词页
- 所有页面 Hugo 静态渲染，确保爬虫完整抓取
- 面包屑导航增强内部链接结构
