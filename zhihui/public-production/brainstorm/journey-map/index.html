用户旅程图是什么
用户旅程图(User Journey Map)以时间线可视化用户的完整使用体验，记录每个触点上用户的行为、想法、情绪，从中识别痛点和改进机会。
怎么使用这个工具
  1. 定义用户画像和使用场景
  2. 划分用户旅程的阶段
  3. 每个阶段填写用户行为、想法和情绪
  4. 标注触点通道（App、网页、线下等）
  5. 识别痛点（低分情绪）并写下改进机会
最佳实践
  • 基于真实用户数据，避免猜测
  • 重点关注情绪曲线的低谷区域
  • 每个触点标注所属渠道，便于优化资源分配
  • 优先解决影响面广、出现频次高的痛点
  • 完成后团队对齐核心洞察，制定行动方案
常见问题
用户旅程图和服务蓝图有什么区别？
用户旅程图从前台用户视角出发，展示用户的感受与体验；服务蓝图则包含后台运营系统、支撑流程等不可见部分。两者常配合使用。
画用户旅程图需要哪些准备？
需要准备用户画像、明确的服务场景、关键触点列表。建议先通过用户访谈或数据分析收集真实用户反馈，避免凭空想象。
旅程图应该包含几个阶段？
通常分为5-8个阶段，如认知→考虑→决策→使用→售后→复购。阶段太少会丢失细节，太多则难以聚焦关键发现。
