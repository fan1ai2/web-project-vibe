# AI Voice — 手绘线稿插画风重设计 Spec

**Date**: 2026-06-15
**Status**: Approved
**Base**: 基于 2026-06-15-ai-voice-transcription-design.md 的视觉重设计

## 设计动机

当前版本存在典型的"AI 工具模板"问题：紫色渐变 Hero + 白底卡片 + 标准 SaaS 布局，缺乏品牌识别度和设计品味。本重设计的目标是做出**让人想用的工具**——有设计感、有情感吸引力、有记忆点。

核心策略：**UI 做减法，插画做表达**。去掉所有"AI 工具"刻板印象元素（紫色、渐变、大面积色块），用手绘线稿插画传递温暖和人性化。

## 字体系统

| 层级 | 字体栈 |
|------|--------|
| 标题 | `"Rounded Mplus 1c", "Noto Sans SC", "Noto Sans", sans-serif` |
| 正文 | `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, "Noto Sans SC", sans-serif` |

## 首页设计

**整体底色** `#FAF7F2` 暖奶油，全页无大面积彩色区块。

### 页面结构

1. **Hero**
   - 大幅内联 SVG 手绘插画（~400×300）：人物坐在桌前，声波纹变成文字
   - 线稿炭黑为主，声波线条 + 文字落点用暖橙 `#E8734A` 点色
   - 插画下方：大字标题 + 一行副标题 + 描边 CTA 按钮
   - 无背景色块、无渐变

2. **步骤插画卡片 ×3**（上传 / 转录 / 导出）
   - 横排三张，无边框无阴影，纯留白区域
   - 每张上方 80×80 SVG 线稿小插画（纯线稿，偶尔暖橙点缀）
   - 下方标题 + 一句话

3. **格式标签**
   - 纯文字标签（逗号分隔），暖灰色小字，无边框无背景
   - 轻到几乎看不见

4. **使用场景 ×5**
   - 横排五张，每张 80×80 SVG 线稿场景插画
   - 下方一行标题，纯线稿无色彩

5. **FAQ**
   - `<details>` 折叠，无容器卡片，无边框
   - Q&A 之间细暖白分隔线
   - 底色保持页面底色

- ~~紫色渐变 Hero~~
- ~~白底阴影卡片系统~~

## 工具页设计

**原则**：去掉一切装饰，干净地做一件事。

### 上传区
- 暖奶油底上微微提亮的白色大圆角区域（无边框）
- 中间 48×48 手绘 SVG：箭头 + 虚线框
- 拖拽悬停：白色底叠加暖黄半透层，箭头变暖橙
- 文字：主提示 + 格式提示（暖灰色更小字号）
- 选中文件后：插画切换为手绘文件图标，显示文件名 + 时长

### 进度指示器
- 四阶段小圆点阵列（识别语言 → 转写 → 时间戳 → 完成）
- 当前阶段暖橙实心，未完成暖白描边空心
- 每 900ms 向前点亮一个
- 文案暖灰色小字在圆点下方

### 音频播放器
- 暗色底（`#1E293B`）保留
- 仅播放按钮 + 进度条 + 时间
- 进度当前位用暖橙圆点标记
- 速度选择器最小化

### 转录面板
- 白色卡片（有细暖白边框，页面唯一带边框的卡片）
- 左侧时间戳暖灰色小字
- 说话人：暖橙圆点 + 炭黑文字
- 当前播放行：炭黑左侧竖线 + 背景微提亮
- 整行点击跳转（不做逐词标记）

### 导出栏
- 四个描边按钮（炭黑边框），悬停 → 实心炭黑底白字

### 隐私标识
- 小字号暖灰文字 + 锁 SVG 线稿图标
- 无背景 badge，无边框

## 插画系统

### 清单

| 位置 | 尺寸 | 着色 |
|------|------|------|
| 首页 Hero | ~400×300 | 线稿 + 局部暖橙 |
| 步骤 ×3 | ~80×80 | 纯线稿 |
| 场景 ×5 | ~80×80 | 纯线稿 |
| 工具页上传 | ~48×48 | 纯线稿，悬停变暖橙 |
| 进度圆点 | CSS 实现 | 暖橙 |
| UI 小图标 | ~16×16 | 纯线稿 |

全部用内联 SVG，不依赖外部资源。

### 手绘风格参数

- `stroke-linecap="round"` + `stroke-linejoin="round"`
- 坐标故意偏离网格 1-2px（模拟手绘不完美）
- 线宽 2px，颜色 `#2D2D2D`
- 暖橙 `#E8734A` 仅用在 1-2 个关键元素
- SVG 设 `viewBox`，百分比宽度，自然响应式
